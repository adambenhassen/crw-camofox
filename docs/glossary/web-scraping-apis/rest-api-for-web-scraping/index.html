REST API for web scraping

Short answer: A REST API for web scraping is an HTTP service that accepts a URL via POST and returns the page's content as JSON. Each call is stateless, the request and response bodies are JSON, and authentication is typically a bearer token in the Authorization header. Any language with an HTTP client can use it without an SDK.

What "REST" means for scraping

REST (Representational State Transfer) is an architectural style where each HTTP method maps to an action on a resource and each request stands alone. For a scraping API the resource is the page you want, the action is POST /v1/scrape (or /extract, /crawl, /map, /search), and the response is JSON containing the extracted content. Statelessness means the server does not remember your previous calls — you pass everything you need in the body each time. This is what makes the API trivial to load-balance, retry, and call from edge functions or background jobs.

Why developers prefer REST scrapers

The alternative is a library-based scraper (BeautifulSoup, Playwright, Scrapy) you embed in your own process. Libraries give you maximum control but require browser binaries, proxy management, and renderer upgrades on every host that runs your code. A REST scraping API hides all of that behind one endpoint. You curl it from a CI job, call it from a Cloudflare Worker, or fan out from a serverless function — the operational surface is one URL and one secret. JSON in, JSON out, language-agnostic.

How fastCRW does it

fastCRW exposes five REST endpoints under https://api.fastcrw.com (hosted) or http://localhost:3000 (self-hosted): /v1/scrape, /v1/crawl, /v1/extract, /v1/map, and /v1/search. All accept JSON bodies, all return {success, data} shapes, and all use Authorization: Bearer $CRW_API_KEY on hosted. Self-hosted requires no auth by default. See the REST API guide for end-to-end usage and the API reference for per-endpoint body schemas. The OpenAPI 3.1 description at /openapi.json is the machine-readable contract.

Frequently asked

What is the difference between a REST scraping API and a scraping library?
A REST scraping API is a hosted HTTP service you call; a library is code you embed in your own process. The API hides browser binaries, proxies, and renderer upgrades; the library gives you direct control at the cost of running infrastructure.
How do I authenticate?
Send Authorization: Bearer $CRW_API_KEY on hosted fastCRW. Self-hosted instances require no auth by default but can be configured to require it.
Is there an OpenAPI spec?
Yes. fastCRW publishes an OpenAPI 3.1 description at /openapi.json so SDK generators, agents, and IDEs can consume the contract directly.
