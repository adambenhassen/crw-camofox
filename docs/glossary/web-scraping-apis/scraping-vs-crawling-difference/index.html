What is the difference between web scraping and web crawling?

Short answer: Web scraping extracts the content of a single, known URL — you give it a link, it returns markdown, HTML, or structured JSON. Web crawling discovers and traverses many URLs starting from one or more seeds, deduplicating links, respecting politeness, and managing a frontier. Scraping is the per-page operation; crawling is the multi-page operation that calls scraping in a loop.

What scraping owns

A scrape is bounded: one HTTP request in, one document out. It decides whether to use a plain HTTP client or a headless browser, runs readability extraction, and returns clean output. Scraping does not concern itself with which URL to fetch next — the caller decides. This makes scraping cheap to reason about: the input is a URL, the output is content, and the cost is one credit (or two if a browser is involved).

What crawling adds

Crawling is scraping plus four extra responsibilities: a frontier (the queue of URLs to visit), dedup (don't re-fetch the same URL twice), politeness (rate limit per host, honour robots.txt), and traversal policy (max depth, max pages, allow-list patterns, BFS vs DFS). Crawling typically runs asynchronously because a site of any size takes minutes to hours to complete, so the API shape is "start a job, poll for status, collect results" rather than a single synchronous call.

How fastCRW does it

fastCRW exposes both operations as separate REST endpoints: /v1/scrape for single-URL extraction (synchronous, returns markdown/HTML/JSON immediately) and /v1/crawl for multi-page traversal (async, returns a job ID you poll). The crawl endpoint accepts maxDepth (cap 10) and maxPages (cap 1000), honours robots.txt by default, and uses the same scrape pipeline under the hood — so a crawl result has the same shape as a scrape result, just many of them. See also /v1/map for URL discovery without content fetching.

Frequently asked

Should I use scraping or crawling for a single product page?
Scraping. If you already know the URL and want the content, /v1/scrape is the right call — it returns synchronously in one request and costs one credit. Crawling adds frontier and traversal overhead you do not need.
Can I crawl an entire site with one scrape call?
No. A scrape call returns content for exactly one URL. To traverse a whole site, use a crawl endpoint, which scrapes each discovered page internally and aggregates the results into a single job.
What is the difference between crawling and mapping?
Mapping discovers URLs on a site without fetching their content — it returns a list of links from sitemaps and lightweight HTML scans. Crawling discovers URLs and fetches each one's content. Mapping is cheaper and faster when you only need the URL inventory.
