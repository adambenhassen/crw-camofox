LLM BYOK extraction explained

Short answer: BYOK ("Bring Your Own Key") extraction means the scraping service calls the LLM with your API key rather than its own. You keep the direct billing relationship with the model vendor, you control which model is used, and your rate limits are the vendor's limits — not a re-sold quota.

Why BYOK matters

When an extraction service bundles LLM usage into its credit system, three things go wrong at scale. First, you pay a markup on every token because the service is reselling the model. Second, you cannot choose your model — you get whatever the service decided to deploy. Third, your effective rate limit is whatever fraction of the service's pooled quota you happen to grab; large jobs throttle unpredictably. BYOK fixes all three: own billing, own rate limits, own model choice. It also makes spend observable in one place (your model vendor dashboard) and lets you negotiate enterprise rates directly.

Trade-offs

BYOK shifts operational responsibility back to the caller. You must rotate keys, monitor model-vendor outages, and set per-key spend caps yourself. For one-off prototypes a bundled-credit model is simpler; for production pipelines BYOK pays for itself within the first thousand calls.

How fastCRW does it

fastCRW's /v1/extract endpoint accepts three BYOK fields in the request body: llmApiKey (your secret key), llmProvider (e.g. openai, anthropic, groq, openrouter), and llmModel (e.g. gpt-4o-mini, claude-haiku-4-5). The key never leaves the request path — it is used once to make the LLM call and is not persisted server-side. Supported providers include OpenAI, Anthropic, Groq, and OpenRouter (which itself fans out to Mistral, Cohere, Together, and others). Minimum 2 credits per extract call cover the orchestration; the LLM cost is billed to your provider account.

Frequently asked

Is my API key stored?
No. The key is used in-flight for the single extract call and is not persisted to disk or logs.
What if I do not provide a key?
Hosted fastCRW falls back to its server-configured provider key when one is set. Self-hosted instances require BYOK or a configured provider key — there is no implicit free tier.
Which provider should I pick?
For cheapest structured extraction at scale, OpenAI gpt-4o-mini or Groq Llama 3.1 8B work well. For complex nested schemas or long pages, larger models (Claude Sonnet, GPT-4.1) reduce retries enough to be cheaper end-to-end.
